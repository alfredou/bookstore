🛍️ BOOKSTORE 
Ecommerce Portfolio Project

Este proyecto es una aplicación de ecommerce desarrollada como parte de mi portfolio personal. El objetivo principal es demostrar habilidades en:

Arquitectura frontend y backend

Manejo de estado

Autenticación básica

Formularios y validaciones

Diseño responsive

Buenas prácticas de desarrollo

⚠️ Nota importante sobre los formularios
📩 Newsletter

El formulario de suscripción a la newsletter no envía información real a ningún servidor ni servicio de email.

La funcionalidad está simulada.

El objetivo es demostrar validación de formularios y flujo de interacción.

En un entorno de producción, se integraría un servicio de email como Resend, SendGrid o similar con dominio verificado.

📬 Formulario de Contacto

El formulario de contacto tampoco envía datos reales.

Los datos no se almacenan ni se transmiten a un backend externo.

La respuesta de éxito es simulada con fines demostrativos.

En un entorno real, este formulario estaría conectado a una API que:

Validaría los datos en backend

Almacenaría la información en base de datos

Enviaría un correo de notificación

🔐 Autenticación

El sistema de login y registro está implementado para demostrar:

Manejo de formularios

Validación de credenciales

Flujo de autenticación

Protección básica de rutas

No se realiza verificación real por email ni envío de códigos OTP.

🎯 Objetivo del proyecto

Este proyecto no está diseñado para producción real, sino como demostración técnica de:

Estructura limpia de código

Separación de responsabilidades

Flujo completo de usuario

Buenas prácticas de desarrollo moderno
